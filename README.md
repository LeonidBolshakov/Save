# Save

[English version](README.en.md)

Save - утилита для резервного копирования файлов в облачное хранилище.

Проект архивирует выбранные файлы и каталоги, защищает архив паролем и отправляет его в облако. Сейчас поддерживаются архиватор 7z и Яндекс.Диск, но структура проекта рассчитана на расширение: можно добавлять другие архиваторы и другие облачные хранилища.

## Назначение

Проект решает две основные задачи:

1. Автоматическое создание защищенного архива и загрузка его в облако.
2. Настройка списка сохраняемых файлов и расписания запуска через графическое окно.

На облачном диске поддерживается структура архивов, чтобы резервные копии не складывались в одну общую папку, а были организованы для последующего поиска и восстановления.

## Состав проекта

### 1. Резервное копирование

Основной модуль резервного копирования расположен в `src/GENERAL/main.py`.

Он выполняет рабочий сценарий:

- проверяет наличие необходимых настроек окружения;
- получает список файлов и каталогов для сохранения;
- создает архив с помощью 7z;
- защищает архив паролем;
- авторизуется в облаке через API Яндекс.Диска;
- загружает архив в облачное хранилище;
- формирует и поддерживает структуру архивов на облачном диске.
- после выполнения сохранения отправляет пользователю e-mail с результатом.

Связанные модули:

- `src/ARCHIVES/` - работа с архиваторами, текущая реализация для 7z;
- `src/YADISK/` - работа с Яндекс.Диском и OAuth;
- `src/GENERAL/` - общая логика резервного копирования, пути, переменные окружения, именование архивов;
- `src/LOGGING/` - настройка логирования;
- `src/MAIL/` - отправка сообщений и уведомлений.

### 2. Настройка сохранения

Графическая программа настройки расположена в `src/SETUP/save_setup.py`.

Она предназначена для подготовки резервного копирования:

- выбирает папки и отдельные файлы, которые нужно сохранять;
- показывает дерево файловой системы с состояниями выбора;
- сохраняет список выбранных путей;
- настраивает задачу в Планировщике заданий Windows;
- позволяет указать дни недели, время запуска, описание задачи, путь к программе и рабочую директорию.

Окно настройки включает две основные области:

- планирование запуска задачи Windows;
- выбор файлов и каталогов для резервного копирования.

## Конфигурация и секреты

Данные, необходимые для работы программы, не должны храниться в репозитории.

Сейчас они передаются сторонними средствами через:

- переменные окружения;
- `keyring`;
- локальные конфигурационные файлы, исключенные из Git.

В `.gitignore` уже исключены чувствительные файлы:

- `*.key`;
- `config_secret.json`;
- `token.json`.

Возможна дальнейшая доработка проекта для автоматизации первичной настройки секретов, токенов и параметров окружения.

## Зависимости

Основные зависимости проекта:

- Python;
- 7-Zip;
- PyQt6;
- pywin32;
- requests;
- yadisk;
- keyring;
- python-dotenv;
- pytest.

Полный список зависимостей находится в `requirements.txt`.

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Также необходимо установить 7-Zip и обеспечить доступ программы к исполняемому файлу архиватора.

## Запуск

Запуск резервного копирования:

```bash
python -m src.GENERAL.main
```

Запуск окна настройки:

```bash
python -m src.SETUP.save_setup
```

## Сборка exe

Для запуска программы из Планировщика заданий Windows необходимо получить исполняемый `exe` файл.

BAT-файл для создания `exe` находится в главной директории проекта:

```bash
create_exe.bat
```

После сборки путь к полученному исполняемому файлу указывается в окне настройки задачи Windows.

## Тестирование

В проекте есть набор тестов для архиватора, OAuth, Яндекс.Диска, планировщика, логирования и общей логики резервного копирования.

